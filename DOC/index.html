Eract - ReactJS Admin Template built on Create-React-App cli from facebook. So, much of the docs are already covered officially. In addition, this version also use SASS  for css styling through the use of custom-react script which I'll describe in this doc and lastly I'll describe the quick summary and structure of this template.

File Structure

The following are the file structure and contents of the template folder:

 ├── Eract/
     ├── app                               : Main folder
         ├── public/                              :  Contains the static assets (index, static images).
         ├── src/                                 :  Contains the development version and core files
             ├── shared/                          : contains shared components (header, nav, footer)
             ├── views/                           : contains all react components and page views.
             ├── App.js, app.scss                 : The wrapper for the whole views and its common style
             ├── index.js, index.scss             : The main starting point of react app and common style.
             ├── routes.js                        : The main routes file which initialize react router.
             ├── .env                             : Used by `custom-react-script` package for supporting sass, less etc.
             ├── package.json                     : The npm (nodejs package manager) file.
             ├── README.md                        : The official create-react-app cli readme.
     ├── doc                               : The DOC you're reading

Quick Start

After you've purchased the template, download the zip file, extract it and do the following:

  • Go to the app/build folder which already compiled to production using webpack build system.
  • Start the http server of your choice (which support HTML5 PushState). If your server doesn't support pushstate, you can install this (pushstate) http server or this server after installing nodejs.
  • Now serve the directory app/build on http-server and open the browser to browse on localhost. That's it. You're now running the production build in the browser.

Start in Development

NOTE: Before you start the template in development, you need to install nodejs (and npm) in your pc.

On top of official create-react-app cli, I use custom-react-script npm package which enables sass or less or css-modules for using with the app. SASS preprocessor already enabled it via .env file which provides custom variables switches to enable/disable preprocessor.

As the template already initiated with the cli, so you don't need to type this command create-react-app my-project-name as given in the official doc. You've to just do the following:

  • Install all the dependencies with npm install by typing in the terminal.
  • And finally type the npm start to start the webpack-dev server.

The dev server will automatically start the browser and listen on localhost:3000 port for development. From that time on, whenever you edit any script or file the server will hot-reload the file and refresh the browser.

Now when you think, your editing will complete and you want the fresh production version. Simply execute npm run build command from terminal to output the production version in app -> build folder.

There's a app -> public folder which include index.html and other static image and favicon. The structure of the index.html is very simple and doesn't do much other than putting google fonts and loading bar.

Using SASS

Beside SASS, you can use also the following preprocessors for css in template -- LESS, Stylus and CSS-Modules. Just enable the variable flag in .env file. And note also if you remove this file, then sass-preprocessor would not work. So, make sure file is there in the main template folder before starting the server.

The SASS (scss) files used in the template is bit different from something we used in simple angular/html templates. I used different sass files for each react components and pages so that each component is independent of each other. Also each component sass files name doesn't start with underscore (_) because it was packed ultimately by webpack. Files which are shared like variables file are start with underscore (_)

Take for eg: the structure of app/src/views/dashboard folder which include specific sass style file for that view and import using es6 import function in js file.

All shared sass files are in the app/src/shared/ folder. I used bootstrap 4 as a base framework and override it's styles and variables in the following files:

  • app -> src -> index.scss
  • app -> src -> shared -> _variables.scss

Using Components

Each react nav views (route views) are divided into components and in separate folder with separate sass stylesheet and attached with react-router in routes.js file. It means you can remove and add each component easily which you don't need. For example, if I want to remove Mail View from the template, I would simply do the following:

  • Will Remove the app -> src -> views -> mail folder
  • Will unlink the mail route from the routes.js file.

Eract, admin template uses bootstrap 4 and reactstrap for ui components. Though optional, reactsrap makes it easy to write bootstrap 4 ui components in react form and easy to readable. Take an example of button component:

<Button color="primary">Simple Btn</Button>  <!-- where color is primary, secondary, warning, info, danger -->

will render this bootstrap primary button with easy to eyes and this is how react make it easy to write component

There's much more than that was already written in the official doc. So head over to it and read how to integrate with your backend, testing components etc.
